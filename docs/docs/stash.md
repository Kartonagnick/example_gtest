Стеш, он же сундук, это - каталог, где размещаются всякие временные файлы,  
которые неизбежно возникают в ходе разработки, и которым не место в репозитории,  
но тем не менее, они нужны для работы.  

Практика показывает, что оптимальная структура каталогов имеет вид:  
```
repo
 |--- _build              сборочные проекты
 |--- _external           результаты сборки внешних библиотек
 |--- _product            результаты сборки нашего проекта
 |--- _repo               скачанные репозитории
 |--- _stash              сундуки
 |     |--- example_gtest   сундук нашего репозитория
  `-- example_gtest       наш репозиторий
```

Идея в том, что имя сундука совпадает с именем репозитория.  
А сам сундук всегда расположен в подкаталоге `_stash`, 
который находится в одном каталоге с основным репозиторием.  
Поэтому, из каталога сундука всегда можно пройти в каталог репозитория.  

```bat
rem определяем имя каталога-сундука
call :getName name "%~dp0."

rem определяем путь к каталогу репозитория
set "path=..\..\%name%"

echo repo.name: %name%
echo repo.path: %path%
exit /b

:getName
  set "%~1=%~n2%~x2"
exit /b
```
<br/>

Зная, где находится основной репозиторий,  
из каталога стеша удобно запускать всякие скрипты.  
Например: скрипт, который будет делать бэкап репозитория.  

Таким образом получается, что стэш, это не только хранилище временных файлов,  
но и местно, где располагаются скрипты для обслуживания репозитория.
