Стеш, он же сундук, это - каталог, где размещаются всякие временные файлы,  
которые неизбежно возникают в ходе разработки, и которым не место в репозитории,  
но тем не менее, они нужны для работы.  

Практика показывает, что оптимальная структура каталогов имеет вид:  
```
repo
 |--- _build              сборочные проекты
 |--- _external           результаты сборки внешних библиотек
 |--- _product            результаты сборки нашего проекта
 |--- _repo               скачанные репозитории
 |--- _stash              сундуки
 |     |--- example_gtest   сундук нашего репозитория
  `-- example_gtest       наш репозиторий
```

Идея в том, что имя сундука совпадает с именем репозитория.  
Поэтому, из каталога сундука всегда можно пройти в каталог репозитория.  

```bat
rem определяем имя каталога-сундука
call :getName name "%~dp0."

rem определяем путь к каталогу репозитория
set "path=..\..\%name%"

echo repo.name: %name%
echo repo.path: %path%
exit /b

:getName
  set "%~1=%~n2%~x2"
exit /b
```
<br/>

Зная, где находится основной репозиторий,  
из каталога стеша удобно запускать всякие скрипты.  
Например: скрипт, который будет делать бэкап репозитория,  
и сохранять архив у себя в каталоге.  

7zrun
---
Батник запускает архиватор 7z и его руками делает бэкапы вида:  
```
repo
 |--- _stash
 |     |--- example_gtest                    
 |     |     |
 |     |     |--- _backup
 |           |     |
 |           |     |--- example_gtest-v001
 |           |     |     |--- [2025-12m-04][12h-35m]-REL.7z 
 |           |     |      `-- [2025-12m-04][14h-10m]-REL.7z
 |           |     |--- example_gtest-v002
 |           |     |     |--- [2025-12m-05][19h-55m]-REL.7z
 |           |     |      `-- [2025-12m-05][20h-15m]-REL.7z
 |           |      `-- example_gtest-v003
 |           |           |--- [2025-12m-06][12h-42m]-REL.7z
 |           |            `-- [2025-12m-06][15h-58m]-REL.7z
 |            `-- 7zrun.bat
 |
  `-- example_gtest
```
Настройки:  
  - `eDEBUG=ON` вкл. отладочный вывод
  - `eDIR_7Z=C:\Program Files\7-Zip` можно указать местоположение архиватора 7z
  - `eDIR_WORK=C:\Program Files\7-Zip` можно указать рабочий каталог
  - `eDIR_SRC=..\repo` можно указать каталог, который нужно заархивировать
  - `eDIR_DST=sample\backup` можно указать каталог, где нужно расположить архив
  - `eNAME7Z=sample` можно указать имя архива
  - `eSTYLE=root-ver\date-pre` можно задать стиль формирования путей для архива
  - `ePASSWORD=**` можно задать пароль для архива
  - `eTIMESPAMP=[2024-12m-22][07h-51m]` можно задать временную метку для формирования путей архива
  - `eEXCLUDE=_*;msvc-temporary;_backup;_video` маски имен каталогов или файлов, которые нужно проигноировать
  - `eCLASSIC=ON` классический режим
  - `eNOSTASH=ON` режим работы "без стеша"
