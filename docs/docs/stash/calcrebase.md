[![P]][H] caclRebase v0.0.1
===========================
Рассмотрим запуск интерактивного ребейза:  
```bat
git rebase -i master~4 --rebase-merges
```

Параметр `-i` означает запуск интерективного режима  
А фрагмент `master~4` означает, что нужно выполнить перебазирование  
для последних 4х коммитов  

Иногда бывает так, что история гита уже достаточно длительная,  
и перебазировать нужно что-то где-то поседине этой истории.  

Батник `caclRebase.bat` помогает вычислить количество коммитов `N`,  
которое нужно указать: `master~N`


Открываем батник на редактирование и находим строки:  
```
version_end = 3
version_beg = 2
delta = version_end - version_beg
result = delta * 2 + 2
```

Здесь:  
  - `version_end` - последняя версия репозитория  
  - `version_beg` - версия бранча, который нужно подправить  

Это работает только при условии соблюдения git-workflow,  
принятого для проектов WorkSpace
